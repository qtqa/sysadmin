LATEST RTA FAILURES DESCRIPTION:

GENERAL
Purpose Improve Release Test Automation (RTA) stability by proving a list of failures in latest test jobs to be able to start needed corrective actions
Question What are the failures (types: ERROR, FAIL, FATAL and XPASS i.e. uneXpected PASS) in each test job
 
METRIC
Filters Test type, License type, Platform
Indicator List of failures in each test job
Calculation Only the failures are listed from the report files, other types (PASS, LOG) are skipped.
The number of PASSes and each failure type are shown per each test job.
Interpretation Zero in failure counts means there were not any failures of affected types
Target value Zero i.e. no failures listed here
 
DATA
Data items Test job name: Job name in Packaging Jenkins, e.g. "Qt5_RTA_opensource_installer_tests_linux_32bit", where "opensource" is a License type, "installer" is a Test type and "linux" is a Platform (as used in the filters)
Job start time: Date and time when job started (shown as it appears in the source result file)
Configuration: One or several per test job (e.g. "linux-g++-Ubuntu11.10-x86")
Installer build number: Number of the offline/online Qt installer under test
Jenkins build history: Running number of test jobs in Packaging Jenkins
Failure description: Failure type and test case name, python script name and line number, and failure description
Summary: Number of PASSed and failed tests per each failure type
Data source Data in result XML files (result_***.xml) saved from Packaging Jenkins to ci-files02-hki.ci.local/home/bld.qt/squishlogs/.
Files are located under each test job directory (e.g. "Qt5_RTA_opensource_installer_tests_linux_32bit") and their Packaging Jenkins build history number directory (e.g. "220") as configuration specific tar.gz files (e.g. "linux-g++-Ubuntu11.10-x86.tar.gz").
Data update cycle New test job result files are saved in real time
